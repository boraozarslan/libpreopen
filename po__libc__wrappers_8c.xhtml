Capability-safe wrapper around the access(2) system call.

access(2) accepts a path argument that can reference the global filesystem namespace. This is not a capability-safe operation, so this wrapper function attempts to look up the path (or a prefix of it) within the current global po_map and converts the call into the capability-safe faccessat(2) if possible. If the current po_map does not contain the sought-after path, this wrapper will call faccessat(AT_FDCWD, original_path, ...), which is the same as the unwrapped access(2) call.
